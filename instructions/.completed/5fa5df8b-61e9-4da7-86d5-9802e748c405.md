# Frontend BFF — Server Actions + Server Components로 Backend API 호출 구조 전환

> **목적**: 브라우저에서 FastAPI backend(`:8000`)를 직접 호출하는 구조를 제거하고,
> Next.js Server Components(읽기) + Server Actions(쓰기)로 전환한다.
> React Query 패키지는 제거하지 않는다 — 인증 레이어 도입 시 활용 예정.

---

## 현재 문제

`frontend/src/lib/api/client.ts`가 다음 구조를 사용한다.

```ts
const API_BASE = process.env.NEXT_PUBLIC_API_URL ?? "http://localhost:8000";
fetch(`${API_BASE}${path}`)
```

- `NEXT_PUBLIC_*` 변수로 backend origin이 브라우저 번들에 노출된다.
- 모든 페이지가 `"use client"` + `useQuery` 구조라 initial fetch도 클라이언트에서 발생한다.
- Docker Compose에서 `http://api:8000` (container internal) vs `http://localhost:8000` (browser) 불일치.

---

## 목표 구조

### 읽기

```text
Server Component (page.tsx)
  -> Server Action / server fetch
  -> FastAPI backend (:8000, server-to-server)
  -> props로 Client Component에 전달
```

### 쓰기

```text
Client Component
  -> Server Action ("use server")
  -> FastAPI backend (:8000, server-to-server)
  -> router.refresh() → Server Component 재실행 → 최신 데이터
```

### 원칙

- `NEXT_PUBLIC_*` 변수에 backend origin을 넣지 않는다.
- `BACKEND_API_URL`은 server-only env var로만 관리한다.
- 페이지 초기 데이터는 Server Component가 fetch한다.
- 필터/페이지네이션 상태는 URL searchParams로 관리한다 (URL 공유 가능).
- mutation 후 상태 갱신은 `router.refresh()`로 처리한다.
- React Query 패키지(`@tanstack/react-query`)는 설치 상태를 유지한다.
  현재 페이지에서는 사용하지 않으나, 인증 도입 시 활용 예정.

## Step 1. server-only fetch 유틸리티 추가

**파일**: `frontend/src/lib/api/server.ts` (신규)

```ts
"use server";

import { ApiError } from "./client";

const BACKEND = process.env.BACKEND_API_URL ?? "http://localhost:8000";

export async function backendFetch<T>(path: string, init?: RequestInit): Promise<T> {
  const res = await fetch(`${BACKEND}${path}`, {
    headers: { "Content-Type": "application/json", ...init?.headers },
    cache: "no-store",
    ...init,
  });
  const body = await res.json();
  if (!body.ok) {
    throw new ApiError(
      body.error?.code ?? "UNKNOWN",
      body.error?.message ?? "Unknown error",
      res.status,
    );
  }
  return body.data as T;
}
```

### 요구사항

- `"use server"` 지시자로 서버에서만 실행됨을 보장한다.
- `cache: "no-store"` — 관리 도구 특성상 항상 최신 데이터를 읽는다.
- `ApiError`는 기존 `client.ts`에서 import한다.

## Step 2. 도메인별 Server Action 파일 추가

**디렉토리**: `frontend/src/lib/actions/`

각 파일 상단에 `"use server"` 를 선언한다. 파일 내 모든 함수가 Server Action이 된다.

### `actions/entities.ts`

```ts
"use server";

import { backendFetch } from "@/lib/api/server";
import type { EntityRead, EntityListResponse, EntityCreate, EntityUpdate } from "@/types/api";

export async function getEntities(qs: string): Promise<EntityListResponse> {
  return backendFetch(`/entities${qs ? `?${qs}` : ""}`);
}

export async function getEntity(id: string): Promise<EntityRead> {
  return backendFetch(`/entities/${id}`);
}

export async function createEntity(body: EntityCreate): Promise<{ id: string }> {
  return backendFetch("/entities", { method: "POST", body: JSON.stringify(body) });
}

export async function updateEntity(id: string, body: EntityUpdate): Promise<EntityRead> {
  return backendFetch(`/entities/${id}`, { method: "PATCH", body: JSON.stringify(body) });
}
```

### 추가 파일 목록

| 파일 | 커버 엔드포인트 |
|------|----------------|
| `actions/aliases.ts` | `POST /entities/{id}/aliases` |
| `actions/contexts.ts` | `POST /entities/{id}/contexts` |
| `actions/relations.ts` | `GET /entities/{id}/relations` |
| `actions/search.ts` | `GET /search` |
| `actions/bundle.ts` | `GET /bundle` |

## Step 3. 페이지를 Server Component + Client Component로 분리

### 패턴

```
app/entities/
  page.tsx           # Server Component — searchParams 받아 fetch
  EntityList.tsx     # Client Component — 필터/페이지네이션 UI, mutation
```

### `app/entities/page.tsx` (Server Component)

```tsx
import { getEntities } from "@/lib/actions/entities";
import { EntityList } from "./EntityList";

export default async function EntitiesPage({
  searchParams,
}: {
  searchParams: Record<string, string>;
}) {
  const qs = new URLSearchParams(searchParams).toString();
  const data = await getEntities(qs);
  return <EntityList data={data} />;
}
```

### `app/entities/EntityList.tsx` (Client Component)

```tsx
"use client";

import { useRouter, useSearchParams } from "next/navigation";
import { useTransition } from "react";

export function EntityList({ data }: { data: EntityListResponse }) {
  const router = useRouter();
  const searchParams = useSearchParams();
  const [isPending, startTransition] = useTransition();

  function setFilter(key: string, value: string) {
    const params = new URLSearchParams(searchParams.toString());
    value ? params.set(key, value) : params.delete(key);
    params.delete("offset"); // 필터 변경 시 첫 페이지로
    router.push(`?${params}`);
  }

  // ...
}
```

### mutation 패턴 (Client Component 내부)

```tsx
const [isPending, startTransition] = useTransition();

function approve() {
  startTransition(async () => {
    await updateEntity(id, { status: "active" });
    router.refresh(); // Server Component 재실행 → 최신 데이터 반영
  });
}
```

### 분리 대상 페이지

| 기존 파일 (전체 Client) | Server Component | Client Component |
|------|------|------|
| `app/entities/page.tsx` | `page.tsx` | `EntityList.tsx` |
| `app/entities/[id]/page.tsx` | `page.tsx` | `EntityDetail.tsx` |
| `app/entities/[id]/edit/page.tsx` | `page.tsx` | `EntityEditForm.tsx` |
| `app/review/page.tsx` | `page.tsx` | `ReviewList.tsx` |
| `app/bundle/page.tsx` | `page.tsx` | `BundleView.tsx` |

### 필터/페이지네이션 상태 관리

- `useState` 대신 URL searchParams 사용
- `router.push(?status=active&offset=20)` → URL 변경 → Server Component 재실행 → 새 데이터
- 새로고침 / URL 공유 시 필터 상태가 유지된다

## Step 4. React Query 훅 파일 제거

현재 `frontend/src/lib/api/` 의 훅 파일들은 `apiFetch` 기반 React Query 훅을 포함한다.
Server Action으로 대체되므로 다음 파일을 제거한다.

- `lib/api/entities.ts`
- `lib/api/aliases.ts`
- `lib/api/contexts.ts`
- `lib/api/relations.ts`
- `lib/api/search.ts`
- `lib/api/bundle.ts`

`@tanstack/react-query` **패키지 자체는 제거하지 않는다.**
`providers.tsx`의 `QueryClientProvider` 설정도 유지한다.

## Step 5. client.ts 정리

**파일**: `frontend/src/lib/api/client.ts`

- `NEXT_PUBLIC_API_URL` 참조 및 `API_BASE` 변수를 제거한다.
- `apiFetch` 함수를 제거한다.
- `ApiError` 클래스는 유지한다 (`server.ts` 및 에러 경계에서 사용).

```ts
// 변경 후 client.ts 전체
export class ApiError extends Error {
  constructor(
    public code: string,
    message: string,
    public status?: number,
  ) {
    super(message);
    this.name = "ApiError";
  }
}
```

## Step 6. 환경 변수 및 Compose 정리

**파일**: `docker-compose.yml`

```yaml
# frontend service — 변경 전
environment:
  - NEXT_PUBLIC_API_URL=http://localhost:8000

# frontend service — 변경 후
environment:
  - BACKEND_API_URL=http://api:8000   # server-only, 브라우저에 노출되지 않음
```

| 변수 | 처리 | 이유 |
|------|------|------|
| `NEXT_PUBLIC_API_URL` | 제거 | 브라우저에 backend origin 노출 |
| `BACKEND_API_URL` | 추가 | Server Action → FastAPI internal URL |

로컬 비컨테이너 개발 시 `BACKEND_API_URL` 미설정이면 `http://localhost:8000` 폴백.
