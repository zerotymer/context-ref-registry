# Project ID 특수문자 제한 — 언더바(`_`)만 허용

## 배경

이전 지침(6dc18568)에서 Project ID 허용 문자를 `A-Za-z0-9_-#^@`로 확장했다.  
그러나 `@`, `#`, `^`, `-` 등 특수문자는 URL 인코딩 문제, CLI 인수 파싱 충돌, 쉘 스크립트 오동작 등을 유발한다.  
**허용 특수문자를 `_`(언더바) 하나로 다시 좁힌다.**

최종 허용 패턴: `^[A-Za-z0-9_]{3,50}$`

---

  ```python
  # Before
  ```

### 3. 테스트 — 기존 확장 허용 케이스 정리

**파일**: `backend/tests/test_project_api.py`

#### 3-a. `test_create_project_valid_extended_ids` 수정

기존 파라미터 중 `_` 외 특수문자가 포함된 값들은 **더 이상 유효하지 않음** → 제거하고, 유효한 케이스로만 구성한다.

```python
# Before — 일부 케이스가 새 규칙에서 422 반환
@pytest.mark.parametrize("project_id", [
    "my-project",           # 하이픈 — 제거
    "team#sub",             # 샵 — 제거
    "v2_api@corp",          # 골뱅이 — 제거
    "ABC123",               # 유지
    "a^b",                  # 캐럿 — 제거
    "my-project_v2#prod@api",  # 복합 특수문자 — 제거
])

# After — 언더바와 알파뉴머릭만
@pytest.mark.parametrize("project_id", [
    "ABC123",
    "my_project",
    "v2_api",
    "team_sub_v2",
    "UPPER_lower_123",
])
```
