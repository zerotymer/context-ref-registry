# Entity 프로젝트 필터 + 번들 그래프 뷰

## 목적

1. **Entity 목록 페이지에 프로젝트별 필터** 추가 — 사용자는 자신이 속한 프로젝트만, ADMIN은 전체 프로젝트를 선택해 Entity를 좁혀볼 수 있다.
2. **번들 탐색기에 그래프 뷰** 추가 — BFS 결과(entities + relations)를 노드-링크 형태의 인터랙티브 그래프로 표시하는 탭을 추가한다.

---

## Feature 1 — Entity 목록 프로젝트 필터

### 현황 분석

- `GET /entities` 는 이미 `AccessPolicy.get_visible_project_ids`로 접근 가능한 프로젝트를 필터링하지만, **query param `project_id`는 없다**.
- `EntityRead` 스키마에 `project_id: str | None` 필드가 이미 있다.
- 프론트엔드 `EntityList.tsx`는 `status`, `types`, `tags` 필터만 제공한다.
- 사용자 가시 프로젝트 목록을 가져오는 엔드포인트: `GET /projects` (본인 소속), `GET /admin/projects` (ADMIN 전용 전체 목록).

### 백엔드 변경

**파일: `backend/app/api/entities.py`**

`list_entities` 함수에 `project_id` query param 추가:

```python
project_id: str | None = Query(None, description="프로젝트 ID로 필터")
```

`EntityService.list()` 호출 시 `project_id` 전달:

```python
items, total = await EntityService(session).list(
    status, types, tags, limit, offset, sort, order,
    visible_project_ids=visible_ids,
    project_id=project_id,      # 추가
)
```

**파일: `backend/app/service/entity_service.py`**

`list()` 메서드에 `project_id: str | None = None` 파라미터 추가.  
WHERE 절에 조건 삽입:

```python
if project_id is not None:
    stmt = stmt.where(Entity.project_id == project_id)
```

단, `project_id`가 `visible_project_ids`에 포함되지 않는 경우는 빈 결과를 반환 (보안).

**테스트: `backend/tests/test_entity_list_api.py`**

- 특정 `project_id` 파라미터로 조회 시 해당 프로젝트 Entity만 반환되는지 검증
- 접근 불가 `project_id`로 조회 시 빈 목록 반환 검증

### 프론트엔드 변경

**파일: `frontend/src/lib/actions/projects.ts` (신규 또는 기존 파일 확인)**

```ts
export async function getMyProjects(): Promise<ProjectRead[]>
```

- 일반 사용자: `GET /api/projects` 호출
- ADMIN: `GET /api/admin/projects` 호출 (현재 사용자 role은 세션/쿠키에서 파악)
- BFF Server Action으로 구현

**파일: `frontend/src/app/(app)/entities/page.tsx`**

- `SearchParams`에 `project_id?: string` 추가
- `buildQs()`에 `project_id` 추가
- `getMyProjects()` 호출하여 프로젝트 목록을 `EntityList`에 prop으로 전달

**파일: `frontend/src/app/(app)/entities/EntityList.tsx`**

- `projects: ProjectRead[]` prop 수신
- 필터 바에 프로젝트 `<select>` 드롭다운 추가:
  - 기본값: "전체 프로젝트"
  - 선택 시 `setFilter("project_id", value)` 호출
- `searchParams.get("project_id")` 로 현재 선택된 프로젝트 표시

**UX 요구사항**:
- 프로젝트가 0개인 사용자(일반 멤버십 없음)는 드롭다운을 표시하지 않는다.
- ADMIN은 모든 프로젝트를 볼 수 있다.
- 프로젝트 변경 시 페이지는 offset=0으로 초기화된다.

## Feature 2 — 번들 탐색기 그래프 뷰

### 현황 분석

- `frontend/src/app/(app)/bundle/page.tsx` — 현재 탭: `context | entities | relations | json`
- Bundle 응답 구조:
  - `entities: BundleEntityRead[]` — `{ id, canonical_name, type, project_id, ... }`
  - `relations: RelationRead[]` — `{ from_entity_id, to_entity_id, relation_type }`
  - `roots: BundleEntityRead[]` — root entity 목록

### 라이브러리 선택: `@xyflow/react` (React Flow)

```bash
pnpm add @xyflow/react
```

- 노드 드래그, 줌/패닝, 미니맵 내장
- 커스텀 노드/엣지 컴포넌트 지원
- SSR-safe (dynamic import 필요)

### 구현

**파일: `frontend/src/components/bundle/BundleGraphView.tsx` (신규)**

```
Props:
  entities: BundleEntityRead[]
  relations: RelationRead[]
  roots: BundleEntityRead[]
```

노드 생성 로직:
- 각 entity → `{ id: entity.id, data: { label: canonical_name, type, isRoot }, position: { x, y } }`
- root entity는 별도 스타일(테두리 굵게, 노란 강조)
- 노드 색상: `ENTITY_TYPE_COLORS` 상수 재사용

엣지 생성 로직:
- 각 relation → `{ id: \`${from}-${to}\`, source: from_entity_id, target: to_entity_id, label: relation_type }`
- 방향 화살표 (`markerEnd: MarkerType.ArrowClosed`)

레이아웃:
- 초기 배치는 간단한 계층 배치 (root를 상단 중앙, BFS depth별로 y 증가)
- 또는 dagre 레이아웃 라이브러리 사용: `pnpm add dagre @types/dagre`

컴포넌트 구조:
```tsx
"use client";
import ReactFlow, { Background, Controls, MiniMap } from "@xyflow/react";
import "@xyflow/react/dist/style.css";
```

SSR 주의: `dynamic(() => import("./BundleGraphView"), { ssr: false })`로 동적 임포트

**파일: `frontend/src/app/(app)/bundle/page.tsx` 수정**

- `ResultTab` 타입에 `"graph"` 추가
- 탭 버튼 목록에 `Graph` 탭 추가: `Graph (${result.relations.length} edges)`
- `resultTab === "graph"` 섹션에 `BundleGraphView` 렌더링

**커스텀 노드 (선택적 강화)**:
- 노드 클릭 시 해당 entity의 `canonical_name`과 `type` 툴팁/패널 표시
- root 노드는 별도 배지(`root`) 표시

### 구현 순서

```
Step 1: 백엔드 — list_entities에 project_id 파라미터 추가 + EntityService 업데이트
Step 2: 백엔드 테스트 추가
Step 3: 프론트 — getMyProjects Server Action 구현
Step 4: 프론트 — EntityList에 프로젝트 필터 드롭다운 연결
Step 5: 번들 그래프 — @xyflow/react + dagre 설치
Step 6: BundleGraphView 컴포넌트 구현
Step 7: bundle/page.tsx에 graph 탭 추가
Step 8: 전체 동작 확인 (docker compose up 기준)
```
