# ISSUE EntityType 추가 및 타입별 CSS 색상 정상화

## 목적

1. `EntityType` 열거형에 `ISSUE` 값 추가
2. Tailwind content 경로 누락으로 인해 프로덕션 빌드에서 타입별 색상 클래스가 제거되는 버그 수정

## 배경 / 원인 분석

### ISSUE 타입

현재 `EntityType` enum(백엔드 `enums.py`, 프론트엔드 `types/api.ts`·`constants.ts`)에
`UI_AREA | FEATURE | INFRA_UNIT | API | CODE_SYMBOL` 5종만 존재한다.
이슈 트래킹 대상(버그, 기획 이슈 등)을 레지스트리에서 직접 참조할 수 있도록 `ISSUE` 타입을 추가한다.

DB 컬럼은 `VARCHAR(50)` + 체크 제약 없음 → **Alembic 마이그레이션 불필요**.

### CSS 색상 누락

`tailwind.config.ts`의 `content` 배열이 아래만 포함한다:

```
./src/pages/**/*.{js,ts,jsx,tsx,mdx}
./src/components/**/*.{js,ts,jsx,tsx,mdx}
./src/app/**/*.{js,ts,jsx,tsx,mdx}
```

`src/lib/constants.ts`는 scan 경로 밖이므로 `ENTITY_TYPE_COLORS`, `ENTITY_TYPE_BAR_COLORS`에
정의된 동적 Tailwind 클래스(e.g., `bg-teal-50 text-teal-700`)가 프로덕션 빌드에서 purge된다.
개발 서버(JIT 전체 포함)에서는 정상처럼 보이지만 빌드 결과물에서 스타일이 사라진다.

## 변경 파일 목록

| 파일 | 변경 내용 |
|------|-----------|
| `backend/app/domain/enums.py` | `EntityType.ISSUE = "ISSUE"` 추가 |
| `backend/app/domain/schemas.py` | `EntityType` 관련 열거 표현 확인 (자동 반영이면 skip) |
| `backend/tests/test_entity_api.py` | ISSUE 타입으로 entity 생성 기본 케이스 1건 추가 |
| `backend/tests/test_ingest_batch.py` | ISSUE 타입 ingest 케이스 추가 |
| `frontend/src/types/api.ts` | `EntityType` union에 `"ISSUE"` 추가 |
| `frontend/src/lib/constants.ts` | `ENTITY_TYPES`, `ENTITY_TYPE_COLORS`, `ENTITY_TYPE_BAR_COLORS`에 ISSUE 항목 추가 |
| `frontend/tailwind.config.ts` | `content` 배열에 `./src/lib/**/*.{js,ts}` 추가 |

## Step-by-Step 구현 지침

### Step 1 — 백엔드 enum 추가

`backend/app/domain/enums.py`의 `EntityType`에 추가:

```python
ISSUE = "ISSUE"
```

순서: 기존 `CODE_SYMBOL` 뒤에 삽입.

### Step 2 — 백엔드 테스트 추가

`test_entity_api.py`에 ISSUE 타입으로 entity를 생성·조회하는 테스트 1건 추가.
기존 패턴(`test_create_entity_*`)과 동일한 구조로 작성.

`test_ingest_batch.py`에 ISSUE 타입을 포함한 ingest payload 테스트 1건 추가.

### Step 3 — 프론트엔드 타입 추가

`frontend/src/types/api.ts`:

```typescript
export type EntityType = "UI_AREA" | "FEATURE" | "INFRA_UNIT" | "API" | "CODE_SYMBOL" | "ISSUE";
```

### Step 4 — 프론트엔드 상수 및 색상 추가

`frontend/src/lib/constants.ts`:

```typescript
export const ENTITY_TYPES: EntityType[] = [
  "UI_AREA", "FEATURE", "INFRA_UNIT", "API", "CODE_SYMBOL", "ISSUE",
];

export const ENTITY_TYPE_COLORS: Record<EntityType, string> = {
  // 기존 5개 유지
  UI_AREA: "bg-indigo-50 text-indigo-700",
  FEATURE: "bg-violet-50 text-violet-700",
  INFRA_UNIT: "bg-teal-50 text-teal-700",
  API: "bg-orange-50 text-orange-700",
  CODE_SYMBOL: "bg-sky-50 text-sky-700",
  // 신규
  ISSUE: "bg-rose-50 text-rose-700",
};

export const ENTITY_TYPE_BAR_COLORS: Record<EntityType, string> = {
  UI_AREA: "bg-indigo-500",
  FEATURE: "bg-violet-400",
  INFRA_UNIT: "bg-teal-400",
  API: "bg-orange-400",
  CODE_SYMBOL: "bg-sky-400",
  // 신규
  ISSUE: "bg-rose-400",
};
```

색상 근거: `ISSUE`는 주의/결함 성격이므로 rose(분홍-빨강 계열) 선택.

### Step 5 — Tailwind content 경로 수정 (CSS 정상화)

`frontend/tailwind.config.ts`의 `content` 배열에 추가:

```typescript
"./src/lib/**/*.{js,ts}",
```

이렇게 하면 `constants.ts`의 동적 클래스 문자열이 Tailwind JIT에 의해 정적 분석되어
프로덕션 빌드에서도 모든 타입별 색상이 유지된다.

### Step 6 — 검증

```bash
# 백엔드 테스트
cd backend && .venv/bin/pytest tests/test_entity_api.py tests/test_ingest_batch.py -v

# 전체 테스트 (271 → 273+ passed 예상)
cd backend && .venv/bin/pytest tests/ -q

# 프론트엔드 타입 체크
cd frontend && pnpm tsc --noEmit
```
