# [BUG] project_admin 계정 프로젝트 관리 화면 접근 오류

## 배경 및 원인

사용자 관리에서 `project_admin` 역할로 생성된 계정이 로그인 후 사이드바의 "내 프로젝트 관리" 메뉴를 클릭하면 403 오류가 발생한다.

**원인 분석:**

사이드바(`Sidebar.tsx`)는 `project_admin` 역할 사용자에게 `/admin/projects` 링크를 노출한다.
`/admin/projects` 페이지(`admin/projects/page.tsx`)도 `project_admin` 역할 사용자의 접근을 허용한다.
그러나 백엔드 `GET /admin/projects`는 `get_current_admin`(role=="admin")만 허용하므로 `project_admin` 사용자가 `listAdminProjects()`를 호출하면 **403 FORBIDDEN**이 발생한다.

```
frontend /admin/projects page  →  GET /admin/projects (admin only → 403)
                                                   ↑
                              project_admin 사용자 접근 차단됨
```

프로젝트 멤버십이 없는 `project_admin` 사용자의 경우 빈 목록이라도 보여야 한다.

## 영향 범위

| 파일 | 위치 | 현재 | 변경 |
|------|------|------|------|
| `backend/app/api/admin_projects.py` | `list_projects` 함수 | `get_current_admin` | `get_current_user` + 역할 기반 필터링 |

## 구현 단계

### Step 1. 백엔드 `GET /admin/projects` 접근 권한 수정

파일: `backend/app/api/admin_projects.py`

`list_projects` 엔드포인트를 수정하여 `admin`은 전체, `project_admin`은 자신의 프로젝트만 조회한다:

```python
from app.auth.dependencies import get_current_admin, get_current_user
from app.repository.project_member_repository import ProjectMemberRepository

@router.get("", response_model=OkResponse[list[ProjectRead]])
async def list_projects(
    session: SessionDep,
    user: Annotated[UserAccount, Depends(get_current_user)],   # admin → user로 변경
    is_active: bool | None = Query(default=None),
    search: str | None = Query(default=None),
) -> OkResponse[list[ProjectRead]]:
    if user.role not in ("admin", "project_admin"):
        raise RegistryError("FORBIDDEN", "Admin or project_admin required", status_code=403)

    all_projects = await ProjectService(session).list_all_projects(
        is_active=is_active, search=search
    )

    if user.role == "admin":
        projects = all_projects
    else:  # project_admin
        member_ids = await ProjectMemberRepository(session).get_project_ids_for_user(user.id)
        member_id_set = set(member_ids)
        projects = [p for p in all_projects if p.id in member_id_set]

    return OkResponse(data=[ProjectRead.model_validate(p) for p in projects])
```

### Step 2. 프론트엔드 — project_admin에게 수정 권한 숨기기 (이미 구현되어 있음)

`ProjectsTable.tsx`의 `isAdmin` prop으로 수정/비활성화 버튼이 이미 제어되고 있다:
```typescript
{isAdmin && (
  <>
    <button onClick={() => setShowEdit(true)}>수정</button>
    <button onClick={toggleActive}>비활성화/활성화</button>
  </>
)}
```

`/admin/projects/page.tsx`에서 `isAdmin={me.role === "admin"}`으로 전달하므로 `project_admin`에게는 읽기 전용으로 표시된다. 이 동작은 유지한다.

### Step 3. 다른 `/admin/projects/*` 엔드포인트 권한 확인

`project_admin` 사용자가 멤버 목록 조회(`GET /admin/projects/{id}/members`)에도 접근할 수 있어야 하는지 검토:

현재 `list_members`, `add_member`, `update_member_role`, `remove_member`는 모두 `get_current_admin`을 사용한다.
**멤버 목록 조회**는 `project_admin`에게도 열어야 한다:

```python
@router.get("/{project_id}/members", response_model=OkResponse[list[MemberRead]])
async def list_members(
    project_id: str,
    session: SessionDep,
    user: Annotated[UserAccount, Depends(get_current_user)],  # admin → user로 변경
) -> OkResponse[list[MemberRead]]:
    if user.role not in ("admin", "project_admin"):
        raise RegistryError("FORBIDDEN", "Admin or project_admin required", status_code=403)
    if user.role == "project_admin":
        member_ids = await ProjectMemberRepository(session).get_project_ids_for_user(user.id)
        if project_id not in member_ids:
            raise RegistryError("FORBIDDEN", "Not a member of this project", status_code=403)
    members = await ProjectService(session).list_members(project_id)
    return OkResponse(data=[MemberRead.model_validate(m) for m in members])
```

멤버 **추가/수정/삭제**는 `admin`만 가능하므로 현재 `get_current_admin` 유지.
