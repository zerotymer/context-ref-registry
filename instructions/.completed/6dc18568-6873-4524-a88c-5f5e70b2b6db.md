# Project ID 제약조건 확장

## 배경

현재 프로젝트 ID는 영문자(A-Za-z) 3~20자만 허용된다.
단어 조합 및 하위 프로젝트 명시를 위해 다음 문자셋이 추가로 필요하다:
- 영소/대문자 + 숫자: `A-Za-z0-9`
- 언더바: `_`
- 하이픈: `-`
- 샾: `#`
- 캐럿: `^`
- 골뱅이: `@`

최대 길이도 20자 → 50자로 확장한다.

## 영향 범위

| 위치 | 파일 | 현재값 | 변경값 |
|------|------|--------|--------|
| 백엔드 regex | `backend/app/service/project_service.py:13` | `r"^[A-Za-z]{3,20}$"` | `r"^[A-Za-z0-9_\-#\^@]{3,50}$"` |
| 백엔드 에러 메시지 | 동 파일 35행 | `"3-20 alphabetic characters"` | `"3-50 characters (A-Za-z0-9_-#^@)"` |
| DB 컬럼 | `backend/app/domain/models.py:30` | `String(20)` | `String(50)` |
| DB FK 컬럼 (4곳) | `models.py:50, 73, 134, 157` | `String(20)` | `String(50)` |
| 프론트엔드 regex | `frontend/src/app/(app)/admin/projects/ProjectsTable.tsx:17` | `/^[A-Za-z]{3,20}$/` | `/^[A-Za-z0-9_\-#\^@]{3,50}$/` |
| 프론트엔드 에러 메시지 | 동 파일 18행 | `"영문자 3~20자만 허용됩니다"` | `"3~50자, 영문·숫자·_·-·#·^·@ 허용"` |
| 프론트엔드 placeholder | 동 파일 44행 | `"영문자 3~20자"` | `"예: my-project, team#sub@v2"` |

## 구현 단계

### Step 1. 백엔드 regex 및 에러 메시지 수정

파일: `backend/app/service/project_service.py`

```python
# 변경 전
_PROJECT_ID_RE = re.compile(r"^[A-Za-z]{3,20}$")

# 변경 후
_PROJECT_ID_RE = re.compile(r"^[A-Za-z0-9_\-#\^@]{3,50}$")
```

에러 메시지도 함께 업데이트:
```python
raise RegistryError(
    "INVALID_PROJECT_ID",
    "Project ID must be 3-50 characters: A-Za-z0-9_-#^@",
    status_code=422,
)
```

### Step 2. DB 컬럼 크기 변경 (Alembic migration)

`backend/` 디렉터리에서:

```bash
.venv/bin/alembic revision -m "extend_project_id_to_50"
```

생성된 migration 파일에 작성:

```python
def upgrade() -> None:
    op.alter_column("project", "id", type_=sa.String(50), existing_type=sa.String(20))
    op.alter_column("project_member", "project_id", type_=sa.String(50), existing_type=sa.String(20))
    # entity 테이블의 project_id FK 컬럼들도 변경
    op.alter_column("entity", "project_id", type_=sa.String(50), existing_type=sa.String(20), existing_nullable=True)
    op.alter_column("entity_metadata", "project_id", type_=sa.String(50), existing_type=sa.String(20), existing_nullable=True)
    op.alter_column("api_key", "project_id", type_=sa.String(50), existing_type=sa.String(20), existing_nullable=True)

def downgrade() -> None:
    op.alter_column("project", "id", type_=sa.String(20), existing_type=sa.String(50))
    op.alter_column("project_member", "project_id", type_=sa.String(20), existing_type=sa.String(50))
    op.alter_column("entity", "project_id", type_=sa.String(20), existing_type=sa.String(50), existing_nullable=True)
    op.alter_column("entity_metadata", "project_id", type_=sa.String(20), existing_type=sa.String(50), existing_nullable=True)
    op.alter_column("api_key", "project_id", type_=sa.String(20), existing_type=sa.String(50), existing_nullable=True)
```

마이그레이션 적용:
```bash
.venv/bin/alembic upgrade head
```

### Step 3. ORM 모델 컬럼 크기 업데이트

파일: `backend/app/domain/models.py`

- `Project.id`: `String(20)` → `String(50)` (line 30)
- `ProjectMember.project_id`: `String(20)` → `String(50)` (line 50)
- `Entity.project_id`: `String(20)` → `String(50)` (line 73)
- (다른 FK 참조 컬럼들도 동일하게 변경)

### Step 4. 프론트엔드 유효성 검사 수정

파일: `frontend/src/app/(app)/admin/projects/ProjectsTable.tsx`

```typescript
// 변경 전 (line 17)

  setError("프로젝트 ID는 영문자 3~20자만 허용됩니다.");

// 변경 후

  setError("프로젝트 ID는 3~50자이며 영문·숫자·_·-·#·^·@만 허용됩니다.");
```

placeholder도 변경:
```typescript

// 변경 후

```
