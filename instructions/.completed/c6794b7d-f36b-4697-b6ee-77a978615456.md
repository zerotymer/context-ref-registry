# Entity 목록 조회 API — GET /entities

> **목적**: 현재 `GET /entities/{id}` (단건)만 있고 목록 조회가 없어
> Review UI 및 Dashboard 구현 시 전체 entity 탐색이 불가능하다.
> `GET /entities` 를 추가해 status/type 필터 + 페이지네이션을 지원한다.

---

## 브랜치

`feat/entity-list-api`

---

## 엔드포인트 스펙

```
GET /entities
  ?status=candidate          # 선택, EntityStatus 단일 값
  &types=UI_AREA&types=FEATURE  # 선택, EntityType 복수
  &limit=20                  # 기본 20, 최대 100
  &offset=0                  # 기본 0
  &sort=created_at           # created_at | updated_at | canonical_name (기본 created_at)
  &order=desc                # asc | desc (기본 desc)
```

### 성공 응답 (200)

```json
{
  "ok": true,
  "data": {
    "items": [ ...EntityRead ],
    "total": 42,
    "limit": 20,
    "offset": 0
  }
}
```

`total`은 필터 조건에 맞는 전체 개수 (페이지와 무관). UI 페이지네이션에 필요.

### Step 2 — Repository (`repository/entity_repository.py`)

- [ ] `list()` 메서드 추가:
  ```python
  async def list(
      self,
      status: EntityStatus | None = None,
      types: list[EntityType] | None = None,
      limit: int = 20,
      offset: int = 0,
      sort: str = "created_at",
      order: str = "desc",
  ) -> tuple[list[Entity], int]:
      ...
  ```
  - 필터 조건 동일한 WHERE 절로 `COUNT(*)` 와 `SELECT` 두 쿼리 실행
  - `sort` 허용값: `created_at`, `updated_at`, `canonical_name`
  - `order` 허용값: `asc`, `desc`
  - 반환: `(items, total)`

  **구현 참고** (SQLAlchemy 2.x async):
  ```python
  from sqlalchemy import func, select
  
  # count
  count_stmt = select(func.count()).select_from(Entity)
  if status:
      count_stmt = count_stmt.where(Entity.status == status)
  if types:
      count_stmt = count_stmt.where(Entity.type.in_(types))
  total = (await self._session.execute(count_stmt)).scalar_one()

  # items
  sort_col = {
      "created_at": Entity.created_at,
      "updated_at": Entity.updated_at,
      "canonical_name": Entity.canonical_name,
  }[sort]
  order_fn = desc if order == "desc" else asc
  stmt = select(Entity)
  if status:
      stmt = stmt.where(Entity.status == status)
  if types:
      stmt = stmt.where(Entity.type.in_(types))
  stmt = stmt.order_by(order_fn(sort_col)).limit(limit).offset(offset)
  items = (await self._session.execute(stmt)).scalars().all()
  
  return list(items), total
  ```

### Step 3 — Service (`service/entity_service.py`)

- [ ] `list()` 메서드 추가:

  async def list(
      self,

### Step 4 — API 라우터 (`api/entities.py`)

- [ ] `GET /entities` 핸들러 추가 (`POST /entities` 위에 위치):
  ```python
  from typing import Literal
  from fastapi import Query
  from app.domain.enums import EntityStatus, EntityType
  from app.domain.schemas import EntityListResponse

  @router.get("", response_model=OkResponse[EntityListResponse])
  async def list_entities(
      session: SessionDep,
      status: EntityStatus | None = Query(None),
      types: list[EntityType] | None = Query(None),
      limit: int = Query(20, ge=1, le=100),
      offset: int = Query(0, ge=0),
      sort: Literal["created_at", "updated_at", "canonical_name"] = Query("created_at"),
      order: Literal["asc", "desc"] = Query("desc"),
  ) -> OkResponse[EntityListResponse]:
      items, total = await EntityService(session).list(status, types, limit, offset, sort, order)
      return OkResponse(data=EntityListResponse(
          items=[EntityRead.model_validate(e) for e in items],
          total=total,
          limit=limit,
          offset=offset,
      ))
  ```

  > **주의**: FastAPI 라우터는 path 패턴 순서로 매칭한다.
  > `GET /entities` 를 `GET /entities/{entity_id}` 보다 **앞에** 등록해야
  > `""` 경로가 `{entity_id}` 로 잘못 매칭되지 않는다.
