# 관리자 콘솔 UI — 로그인·사용자·프로젝트·멤버 관리 화면

> **목표**: 전역 관리자와 프로젝트 관리자가 웹 UI를 통해 사용자·프로젝트·팀원을 관리할 수 있게 한다.
> 목업(`mockups/`) 기준으로 구현하며, 목업은 `http://localhost:8080`에서 확인 가능하다.

---

## 확정 설계 결정

| 항목 | 결정 |
|------|------|
| 로그인 방법 | 이메일 + 비밀번호만 제공 |
| API Key 로그인 UI | **제외** — API Key는 REST API `Authorization: Bearer` 전용 |
| 인증 쿠키 | `access_token` (httpOnly, JWT) |
| 사이드바 관리자 섹션 | `admin`만 노출 — 사용자 관리, 프로젝트 관리 |
| 사이드바 프로젝트 섹션 | `project_admin`만 노출 — 멤버 관리 |
| `user` 역할 | 관리자 관련 메뉴 완전 비노출 |
| 현재 사용자 표시 | 사이드바 하단 — 이메일 + 역할 뱃지 |
| 비활성 항목 | `opacity-60` 처리, 기능은 비활성 |
| 프로젝트 관리자 변경 | `admin` 전용 — `project_admin` 화면에서 버튼 `disabled` |
| 생성 액션 | 모달로 처리 (페이지 이동 없음) |
| 테이블 행 액션 | ⋮ 드롭다운 메뉴 |

### 작업 목록

- [ ] `/login` 페이지 생성 (Next.js App Router)
- [ ] 이메일 + 비밀번호 폼 구현
- [ ] `POST /auth/login` 호출 → `access_token` 쿠키 수신
- [ ] 로그인 실패 에러 메시지 표시 (이메일/비밀번호 불일치)
- [ ] 비활성 계정 에러 메시지 표시
- [ ] 로그인 성공 시 `/` 리다이렉트

### 완료 조건

- 유효한 계정으로 로그인하면 관리자 대시보드로 이동한다.
- 잘못된 자격증명은 에러 메시지를 표시한다.
- 비활성 계정은 별도 메시지로 안내한다.
- 화면에 API Key 입력란이 없다.

## Step 2-3-UI-2. 공통 레이아웃 — 사이드바 권한 분기

**브랜치**: `feat/step2-3-ui-layout`
**상태**: `[ ]` pending

### 작업 목록

- [ ] `GET /auth/me` 호출로 현재 사용자 role 조회
- [ ] 사이드바 구성 규칙 구현
  | role | 노출 메뉴 |
  |------|-----------|
  | `admin` | Dashboard, Entity 목록, 승인 대기, Bundle 탐색기 + **관리자** (사용자 관리, 프로젝트 관리) |
  | `project_admin` | Dashboard, Entity 목록, 승인 대기 + **내 프로젝트** (멤버 관리) |
  | `user` | Dashboard, Entity 목록, 승인 대기, Bundle 탐색기 |
- [ ] 사이드바 하단 현재 사용자 이메일 + 역할 뱃지 표시
  - `admin` → 빨간 뱃지
  - `project_admin` → 인디고 뱃지
  - `user` → 회색 뱃지
- [ ] 로그아웃 버튼 (`POST /auth/logout` → `/login` 리다이렉트)
- [ ] 권한 없는 경로 접근 시 403 또는 `/` 리다이렉트

### 완료 조건

- `user`는 관리자 메뉴를 볼 수 없다.
- `project_admin`은 사용자·프로젝트 관리 메뉴를 볼 수 없다.
- `admin`은 모든 메뉴를 볼 수 있다.

**완료일**: —
