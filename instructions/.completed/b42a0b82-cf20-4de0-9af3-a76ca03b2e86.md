# [BUG] 승인 대기 Entity 승인 불가 및 오류 표시 누락

## 배경 및 원인

승인 대기 목록(review 페이지)에서 "✓ Active 승인" 버튼 클릭 시 두 가지 문제가 있다:

### 문제 1: API 실패 시 오류가 사용자에게 표시되지 않음

`ReviewList.tsx`의 `approve()` 함수:

```javascript
function approve() {
  onDismiss();  // ← API 호출 전에 먼저 UI에서 제거
  startTransition(async () => {
    await updateEntity(entity.id, { status: "active" });  // 실패해도 에러 표시 없음
    router.refresh();
  });
}
```

`onDismiss()`가 API 호출 **전**에 실행되므로:
- API가 403/500 등으로 실패해도 entity는 UI에서 사라짐
- `startTransition` 내부에서 throw된 에러는 사용자에게 표시되지 않음
- 페이지를 새로고침하면 entity가 다시 나타남 (DB에는 candidate 상태 유지)

### 문제 2: 비-admin 사용자는 entity를 승인할 수 없음

`PATCH /entities/{id}` 권한 로직(`policy.check_can_mutate_entity`):
- `admin` 역할: 모든 entity 수정 가능
- 그 외: `project_id`가 없는 public entity → 403 ("Only admin can modify public entities")
- 그 외: `project_id`가 있어도 해당 프로젝트 멤버가 아니면 → 403

admin이 아닌 사용자가 review 페이지에서 승인을 시도하면 403이 발생하지만 UI에 표시되지 않아 "승인 불가 버그"처럼 보인다.

## 영향 범위

| 파일 | 문제 |
|------|------|
| `frontend/src/app/(app)/review/ReviewList.tsx` | `approve()`, `archive()` 함수 - 에러 처리 없음, onDismiss 순서 오류 |

## 구현 단계

### Step 1. `approve()` / `archive()` 에러 처리 추가

파일: `frontend/src/app/(app)/review/ReviewList.tsx`

`ReviewCard` 컴포넌트에 에러 상태를 추가하고, onDismiss를 성공 후로 이동:

```typescript
function ReviewCard({
  item,
  onDismiss,
}: {
  item: ReviewItem;
  onDismiss: () => void;
}) {
  const { entity, aliases, contexts, relations } = item;
  const router = useRouter();
  const [isPending, startTransition] = useTransition();
  const [error, setError] = useState<string | null>(null);  // 추가
  ...

  function approve() {
    setError(null);
    startTransition(async () => {
      try {
        await updateEntity(entity.id, { status: "active" });
        onDismiss();   // ← 성공 후로 이동
        router.refresh();
      } catch (err: unknown) {
        setError(err instanceof Error ? err.message : "승인에 실패했습니다.");
      }
    });
  }

  function archive() {
    setError(null);
    startTransition(async () => {
      try {
        await updateEntity(entity.id, { status: "archived" });
        onDismiss();   // ← 성공 후로 이동
        router.refresh();
      } catch (err: unknown) {
        setError(err instanceof Error ? err.message : "보류 처리에 실패했습니다.");
      }
    });
  }
```

카드 하단에 에러 메시지 표시 영역 추가:

```typescript
{error && (
  <div className="mt-2 px-3 py-2 bg-red-50 border border-red-200 rounded text-xs text-red-700">
    {error}
  </div>
)}
```

### Step 2. 승인 권한이 없는 사용자에게 버튼 비활성화 또는 권한 안내

review 페이지(`review/page.tsx`)에서 현재 사용자 정보를 가져와 ReviewList에 전달:

```typescript
// review/page.tsx
import { getCurrentUser } from "@/lib/actions/auth";

export default async function ReviewPage() {
  const [me, data] = await Promise.all([
    getCurrentUser(),
    getEntities("status=candidate&limit=50&sort=created_at&order=asc"),
  ]);

  const items: ReviewItem[] = await Promise.all(
    data.items.map(async (entity) => { ... })
  );

  return <ReviewList items={items} canApprove={me?.role === "admin"} />;
}
```

`ReviewList` 및 `ReviewCard`에 `canApprove` prop 추가:

```typescript
// ReviewList
export function ReviewList({ items, canApprove }: { items: ReviewItem[]; canApprove: boolean }) {
  ...
  return (
    <>
      ...
      {visible.map((item) => (
        <ReviewCard
          key={item.entity.id}
          item={item}
          canApprove={canApprove}
          onDismiss={...}
        />
      ))}
    </>
  );
}

// ReviewCard - approve 버튼에 조건 추가
<button
  onClick={approve}
  disabled={isPending || !canApprove}
  title={!canApprove ? "admin 권한이 필요합니다" : undefined}
  className="text-xs px-4 py-1.5 rounded-md bg-green-600 text-white font-medium hover:bg-green-700 disabled:opacity-60 disabled:cursor-not-allowed"
>
  ✓ Active 승인
</button>
```

> **참고**: 현재 구현에서 public entity(project_id=null)는 admin만 수정 가능하고,
> project entity는 해당 프로젝트 멤버만 수정 가능하다.
> 간단한 처리로 admin만 승인 버튼을 활성화한다.
> 추후 entity의 project_id 기반으로 세분화할 수 있다.

### Step 3. 에러 메시지 포맷 정리

`backendFetch`에서 throw되는 `ApiError`의 메시지를 사용자 친화적으로 표시:

`ApiError` 클래스 확인 (`frontend/src/lib/api/client.ts` 등):
```typescript
export class ApiError extends Error {
  constructor(public code: string, message: string, public status: number) {
    super(message);
  }
}
```

에러 표시 시 코드와 메시지를 함께 표시:
```typescript
} catch (err: unknown) {
  if (err instanceof ApiError) {
    setError(`[${err.code}] ${err.message}`);
  } else {
    setError(err instanceof Error ? err.message : "승인에 실패했습니다.");
  }
}
```

### Step 4. 테스트

프론트엔드 UI 수동 테스트 시나리오:

1. **admin 계정으로 로그인** → 승인 버튼 활성화, 클릭 시 entity가 active로 변경되고 목록에서 제거
2. **non-admin 계정으로 로그인** → 승인 버튼 비활성화, 툴팁으로 "admin 권한 필요" 표시
3. **네트워크 오류 시뮬레이션** → 카드 하단에 에러 메시지 표시, entity가 목록에서 제거되지 않음
4. **승인 후 페이지 새로고침** → 승인된 entity가 목록에 없음 (active로 변경됨)
